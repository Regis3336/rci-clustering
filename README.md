## 📖 What is RCI?

RCI generates canonical cells, canonical radii, and canonical homology. A nontrivial Čech nerve is not the primary output; Čech nerves appear only as scale-sections of the RCI nerve moduli during radial scale exploration.

Reverse Clustering Impact (RCI) is a metric radial procedure that transforms a finite metric space into locally calibrated canonical cells.

Each RCI cell is generated by an algorithmic center, a local radial Morse profile, a boundary index, and a canonical local radius \(r_j^\star\). The primary output is not merely a set of labels: it is a canonical cell system carrying intrinsic geometric and homological information.

RCI is not spectral clustering, density clustering, or graph clustering. Spectral embeddings and matrix-on-demand Laplacians are computational workspaces; the mathematical core is the metric radial construction.

At the canonical level, RCI outputs:

- canonical cells;
- canonical radii;
- a canonical partition / cell assignment;
- canonical homology.

A nontrivial Čech nerve is not a primary output. It appears only after radial scale exploration, when canonical cells are thickened or varied across scale. The resulting scale-indexed family of nerves is the **RCI nerve moduli**.

RCI generates its local canonical radii internally. A global scale parameter is used to navigate the moduli of gluings among the resulting cells.

---

## 🧠 Core ideas

**Canonical RCI Cells** — RCI does not merely assign labels. It constructs locally calibrated metric cells, each with an algorithmic center, a radial Morse profile, a boundary index, and a canonical radius \(r_j^\star\).

**Canonical Homology** — the canonical RCI output already carries homological information through its intrinsic cell system and local radii.

**Nerve Moduli** — RCI does not output a nontrivial Čech nerve at the primary level. Čech nerves appear only as scale-sections \(N(\lambda)\) of the RCI nerve moduli during radial scale exploration.

**Curvature Law** — cell boundaries are detected via the curvature-sensitive signature \(\Delta^2 M_c(k)\), which is negative on positively curved regions and positive on saddles.

**Matrix-on-Demand Spectral Engine** — the `MatrixOnDemandLaplacian` provides a scalable computational workspace without constructing dense \(N \times N\) matrices. It supports RCI but does not define it.

**Morse Erasure Index (MEI)** — a parameter-free intrinsic metric that quantifies how much of the Morse density field persists across cell boundaries.

## 🧩 Implementation and theory

The code is a **computational instantiation** of the mathematical framework, not a heuristic approximation.

The `SpectralRCI` class name is historical: the reported benchmark configuration can run with `use_spectral=False`, and the formal RCI mechanism is metric. Spectral embeddings and matrix-on-demand Laplacians are computational workspaces used when appropriate.

The implementation validates the canonical cell system, canonical radii, MEI behavior, and the dynamic homology / nerve-moduli layer through reproducible scripts.
